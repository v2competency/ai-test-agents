# Manual Tests Directory

> Output directory for manual test cases generated by the Screenshot Analyzer Agent

---

## Purpose

This directory stores all manual test case documents generated by the Screenshot Analyzer Agent. These files serve as:

1. **Human-readable test documentation** for manual testing
2. **Input for the Automation Generator Agent** to create Playwright automation

---

## File Naming Convention

```
{ApplicationName}_Manual_Test_Cases.txt
```

Examples:
- `SampleApp_Manual_Test_Cases.txt`
- `ECommercePlatform_Manual_Test_Cases.txt`
- `TaskMasterPro_Manual_Test_Cases.txt`

---

## File Structure

Each generated file follows this structure:

```
================================================================================
                    {APPLICATION_NAME} - MANUAL TEST CASES
================================================================================
Application URL: {URL}
Document Version: {version}
Created Date: {date}
Generated By: Screenshot Analyzer Agent v1.0

================================================================================
                              TEST SUMMARY
================================================================================
[Module breakdown and counts]

================================================================================
                        MODULE: {MODULE_NAME}
================================================================================

------------------------------------------------------------------------------
TC_{MODULE}_{###}: {Test Case Title}
------------------------------------------------------------------------------
Test Type: {Positive|Negative|Boundary|Security|E2E}
Priority: {High|Medium|Low}
Precondition: {state}

Test Data:
- field: value

Steps:
1. {action}
2. {action}

Expected Result:
- {outcome}

------------------------------------------------------------------------------
[More test cases...]

================================================================================
                         END OF TEST CASES
================================================================================
```

## Test ID Reference

| Prefix | Module |
|--------|--------|
| `TC_AUTH_` | Authentication |
| `TC_REG_` | Registration |
| `TC_DASH_` | Dashboard |
| `TC_FORM_` | Forms |
| `TC_LIST_` | Lists/Tables |
| `TC_CRUD_` | CRUD Operations |
| `TC_NAV_` | Navigation |
| `TC_SEARCH_` | Search |
| `TC_CART_` | Shopping Cart |
| `TC_CHKOUT_` | Checkout |
| `TC_SEC_` | Security |
| `E2E_` | End-to-End |

### Number Ranges

| Range | Test Type |
|-------|-----------|
| 001-099 | Positive |
| 100-199 | Negative |
| 200-299 | Empty Fields |
| 300-399 | Boundary |
| 400-499 | Security |

## Usage

### For Manual Testing
1. Open the test case file for your application
2. Execute test cases in order or by priority
3. Record pass/fail status
4. Document any deviations

### For Automation Generation
1. Ensure test case file is in this directory
2. Run the Automation Generator Agent with the file path
3. Generated Playwright tests will reference these test IDs
