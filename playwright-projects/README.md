# Playwright Projects Directory

> Output directory for Playwright automation frameworks generated by the Automation Generator Agent

---

## Purpose

This directory stores all Playwright automation projects generated from manual test cases. Each project is a complete, ready-to-run test automation framework.

---

## Project Naming Convention

```
{ApplicationName}/
```

Examples:
- `SampleApp/`
- `ECommercePlatform/`
- `TaskMasterPro/`

---

## Pipeline Position

```
[Screenshots] → [Screenshot Analyzer Agent] → [Manual Tests] → [Automation Generator Agent] → [playwright-projects/]
                                                                                                        ↑
                                                                                                  (This Directory)
```

---

## Generated Project Structure

Each generated project follows this structure:

```
{AppName}/
├── pages/                    # Page Object Model classes
│   ├── BasePage.ts          # Base class with common methods
│   └── {Page}Page.ts        # Page-specific classes
├── tests/                    # Test specifications
│   ├── {module}.spec.ts     # Module tests
│   └── e2e/                 # End-to-end tests
├── data/                     # Test data (JSON)
│   └── {module}Data.json    # Module test data
├── utils/                    # Utilities
│   ├── SelfHealingLocator.ts # Self-healing engine
│   └── AIObserver.ts        # AI element detection
├── playwright.config.ts      # Playwright configuration
├── tsconfig.json            # TypeScript configuration
├── package.json             # Dependencies
├── .env.example             # Environment template
└── README.md                # Project documentation
```

---

## Quick Start for Any Project

```bash
cd playwright-projects/{AppName}

# Install dependencies
npm install

# Install browsers
npx playwright install

# Configure environment
cp .env.example .env
# Edit .env with your values

# Run tests
npm test
```
